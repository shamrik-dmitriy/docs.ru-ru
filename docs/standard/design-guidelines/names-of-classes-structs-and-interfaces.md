# <a name="names-of-classes-structs-and-interfaces"></a>Имена классов, структур и интерфейсов
Руководящие принципы именования, которые следует, применять к общему наименованию типов.

 ✔️ задавайте имена классов и структур с существительными или именными фразами, используя PascalCasing.

 Это отличает имена типов от методов, которые называются глагольных фразами.

 ✔️ задавайте имена интерфейсам с прилагательными или иногда с существительными или именными фразами.

 Существительные и словосочетания должны использоваться редко, и они могут указывать на то, что тип должен быть абстрактным классом, а не интерфейсом.

 ❌ не присваивайте названиям классов префикс (например, "C").

 ✔️ учитывайте, что следует заканчивать имена проихводных классов именем базового класса.

 Это очень легко читается и чётко объясняет отношения. Некоторые примеры этого в коде `ArgumentOutOfRangeException`, который является разновидностью `Exception`, и `SerializableAttribute` который является разновидностью `Attribute`. Однако при применении этого руководства важно использовать разумные суждения. Например, класс `Button` является своего `Control` рода событием `Control`, хотя и не отображается в его названии.

 ✔️ следует добавлять перфикс I к именам интерфейсов, чтобы указать, что тип является интерфейсом.

 Например, `IComponent` (описательное существительное), `ICustomAttributeProvider` (существительное), и `IPersistable` (прилагательное) являются подходящими именами интерфейса. Как и в других именах типов, избегайте сокращений.

 ✔️ необходимо убедиться, что имена отличаются только префиксом "I" в названии интерфейса при определении пары класс-интерфейс, где класс является стандартной реализацией интерфейса.

## <a name="names-of-generic-type-parameters"></a>Названия параметров общего типа
 Обобщения были добавлены в .NET Framework 2.0. Функция представила новый вид идентификатора, называемый *параметром типа.*

 ✔️ необходимо присваивать описательные имена общим параметрам типа, если только однобуквенное имя не явлется полностью понятным и описательное имя не добавит значения. 
 
 ✔️ следует учитывать, что необходимо использовать символ `T` в качестве имени параметра типа для типов с одним однобуквенным параметром типа.

```csharp
public int IComparer<T> { ... }
public delegate bool Predicate<T>(T item);
public struct Nullable<T> where T:struct { ... }
```

 ✔️ в именах общих типов следует использовать префикс `T`.

```csharp
public interface ISessionChannel<TSession> where TSession : ISession {
    TSession Session { get; }
}
```

 ✔️ расммотрите возможность, указания ограничений, наложенных на параметр типа в названии параметра.

 Например, параметр, ограниченный `ISession` модет называться `TSession`.

## <a name="naming-enumerations"></a>Именование перечислений
 Названия типов перечислений (также называемые перечислениями) в целом должны следовать стандартным правилам именования типов (PascalCasing и т.д.). Однако существуют дополнительные рекомендации, которые применяются конкретно к перечислениям.

 ✔️ используйте имя единственного числа для перечисления, если его значения не являются битовыми полями.

 ✔️ используйте имя множественного числа для перечисления с битовыми полями в качестве значений, также называемых флагами enum.

 ❌ не используйте суффикс "Enum" в именах типов enum.

 ❌ не используйте суффиксы "Flag" или "Flags" в именах типов enum.

 ❌ не используйте префикс для имен значений перечисления (например, "ad" для перечислений ADO, "rtf" для перечислений расширенного текста и т.д.).
